Tips

Here are some other handy things I've found to be useful on Unix based systems over the years. There's plenty more, but I'm only bothered to write so much.

xargs

xargs is a command that, given standard output, converts words into program arguments. This is useful for many reasons, such as piping a file into a command as arguments. Examples are provided below.

# fun recursive example
$ echo 1 2 3 4 | xargs echo
# force yourself to a limited number of arguments
$ echo 1 2 3 4 | xargs -n 2 echo
# now compress an entire directory
$ find -R . | xargs tar zcvf file.tar
# which, of course, can also be done as "tar zcvf file.tar *"

It can also do special things like deal with blanks or special characters (it is more useful than the backslashes in this way). You can find more info here .

apt-search

You may know the use of this as sudo apt-get install surprises (or some sort of surprise). But sometimes you don't know what surprise to install.

Introducing apt-cache search

$ apt-cache search surprises

tar

You will be required to use tar to untar files many times in your life. Here's the two ways of using them:

You will need to know the following flags:

Flag Use
z Using the gzip program to archive files. Most popular with linux. You may need to use j
j Zips in the bzip2 format. Better compression at slower speeds.
v Verbose. As you should know by now, it means displays exactly what it's doing (i.e. what is being compressed or decompressed)
f File name you are to archive to/from
c Compress. This compresses the files.
x Extract. Decompresses the tar archive.

Compress

For this example, we will use gzip (as UNIX do).

# syntax: tar zcvf [tarname] [files/directories]
$ tar zcvf lol.tar /path/to/file sample.html

Decompress

# syntax: tar zxvf [tarname]
$ tar zcvf lol.tar

Installing from Source

Installing specific versions of the program required (i.e. not the one in sudo apt-get install means you need to learn to install from source. This means getting the source of the code, compiling it yourself, then installing it. Below is a simplification of the usual steps required.

# in a directory of your choice
$ wget http://example.com/tarball.tar
# install progress will be shown

# now we untar it
$ tar zxvf tarball.tar
# progress of tar is shown

# now we go to the directory
$ cd tarball

# next, we configure, with commands as necessary
$ ./configure
# sometimes, you may need ./configure --setting=xavc - read the manual
# sometimes, you will need to install extra things. But you can use sudo apt-cache search
# this can take a while, and a lot of text appears

# then we "make" it. A makefile (named "Makefile") has instructions to compile it
$ make
# this can take a while - we are compiling the entire program! a lot of textual vomit is shown

# finally, after it compiles, we install it
$ make install
# a lot of vomit is spat, but at last, the program is installed
