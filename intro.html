Introduction

Introduction

Hello and welcome to the Beginner's Guide of the UNIXverse! Today we will cover useful features available in shell - through the website content, talks and exercises (and hopefully, by the time we finished this, with solutions.

But first of all, what is shell? Many of you may think that the terminal you use (xterm, terminator, etc.) is a shell - you are partially right - it is an interactive shell. A shell is basically an interface that allows you to interact with a system's interface. The reason terminals you are used to are interactive is because you get to...well...interact with it (you give it input, they give output/errors, you give it more inputs [or rage quit].

One important thing to note about shells is...well, they are a "shell". All commands within a shell are self-contained - this will be an important concept in shell scripting.

Cryptic Symbols

When we are interacting with the terminal (or you are viewing commands online), you commonly see symbols like $, ~ and % (and most probably a bunch of text) before commands. This is merely to indicate that the command is entered by the user, and is not the output of a program. Below is a demonstration of what you may see.

                # this is a comment
                $ echo "you may see it like this"
                you may see it like this
                Oliver $ echo "or like this, with Oliver in front"
                or like this, with Oliver in front
                Oliver % echo "or this"
                or this
A common newbie mistake when using shell is to include the preceding text before commands when copying something from the shell. Please don't do this, as it only leads to confusing errors and CHAOS! Like that '$' just after Oliver, that's not a part of the echo command, so it shouldn't be copied.

Basic Commands

These are some of the most basic and core commands avaliable in basically every variation of Linux/Unix ever.

Command Description
cd
Stands for: Change Directory 		A directory is the formal name for a folder. The first thing that you need to know about cd is that UNIX sorts directories in a hierarchy, which means that all files and directories have a 'parent' directory (the directory above the current one), and all directories can have 'children', which are either files or directories. cd is the tool used to change from one directory to another. Think about this like the folders in Windows Explorer - double-clicking on a folder opens it and shows its contents (its children). Examples:
  • cd ~ - takes you to your home directory
  • cd folder - if there is a directory called "folder" in the current directory, moves into that directory.
  • cd .. - goes into the parent folder.
  • cd /, which goes to the root directory. The root directory is the only directory that doesn't have a parent. Every directory and file in your file system is therefore a child of the root directory, or a grandchild, or ...
mkdir
Stands for: Make Directory 		Makes a new directory as a child of the current directory.
e.g. mkdir beans
ls
Stands for: List 		Lists all files in the current directory. All of these commands have options. While ls provides just file and directory names, ls -la produces a vertical list with the details of all files, including hidden files. This can be incredibly useful!
rm
Stands for: Remove 		Deletes files or directories. May require additional options to delete folders and protected files.
rm -r recursively deletes a folder, so it deletes all contents of the folder first, and then the folder itself.
rm -f force the deletion of something. rm can take multiple arguments, so you can delete more than one thing at a time.
For example: rm file1 file2 will delete the files named "file1" and "file2".
echo echo echoes (repeats) whatever arguments you give it to standard output (the screen), followed by a new line.
Use:
echo "Hello World!"
If your arguments contains spaces, or special characters (like *, ?), you can use quote marks to escape them, e.g. cd "lol* special folder" (although avoid using spaces in names).

Shell Shortcuts

Here are a few (useful) shortcuts to shell.

Reuse previous command

Hitting the "up" key will automatically place a previously executed command in your next command to execute. This works up until you reach the earliest point of history that the computer records.

Going down, of course, does the opposite if you need it

Tab Completion

Essentially completes a command you have half-typed out so you don't have to waste precious milliseconds writing the rest of it. It is literally magical.

Mess around with the command above by making directories and navigating through them so you get used to it all. Make a folder with a really long name and then tab-complete the name when you cd into it. An example might be:

                    mkdir antidisestablishmentarianism
                    cd anti<tab> <enter>

Try it, your life is now measurably better.

Keyboard Shortcuts

There are a number of Keyboard Shortcuts that are handy in a terminal, including:

Shortcut Description
Alt+Backspace or CTRL+W Deletes the previous word you are on
CTRL+A Go to beginning of line
CTRL+E Goes to end of line
CTRL+L Clears the screen - similar to clear
CTRL+U Clear up anything before the cursor
CTRL+K Clears anything after the cursor
CTRL+R Type in a command and it'll go through your command history. Sometimes this is a better option than holding down the Up key to get to a previous instruction.
CTRL+RIGHT or ALT+F Move forward one word
CTRL+LEFT or ALT+B Move backwards one word

Standard Streams

This workshop sometimes relies on you knowing what the standard streams are - standard input (stdin), standard output (stdout) and standard error (stderr). They are further described below:

Standard Input

This is where a program reads information from - input is provided by the user, or by redirection. It is also known as stdin. C reads from stdin using scanf in C, and Python uses raw_input().

Standard Output

This is where data is usually printed out from a program. This information is usually displayed in your terminal, unless it is redirected. This is also known as stdout. To print to stdout, C uses printf in C, and Python uses print.

Standard Error

This is where errors are usually printed out from a program. It is also known as stderr. C prints to stderr using fprintf(STDERR, "");, and Python prints to stderr using sys.stderr.write.

Redirection

You may sometimes want to change where the standard streams go or come from - for example, putting all messages printed to stderr into a file.

<

Redirects the contents of a file into stdin, for a program to read. Example: ./program < information.txt pushes the contents of information.txt into stdin, program to read.

>

Redirects stdout from a program to a file. Example: ./program > output.txt puts the output of my program into output.txt.

You can use >> to append to the end of a file, as > rewrites the entire file.

Note you can also combine redirectors: ./program < in.txt > out.txt.

Note you can use > /dev/null, the "data sink", which is the equivalent of throwing output down the drain. This is useful for redirecting information that you don't want to see without having to put that output into a file.

2>

This does the same redirection, but for standard error. For example, ./program 2> errors.txt logs errors from the program into error.txt.

You can redirect multiple output streams at once, but this can be messy and complicated. See this for more information.

|

This is a pipe, which takes the standard output of the left hand side of the pipe and "pipes" that output to standard input of the command on the right hand side of the pipe. This is very important in the next section, UNIX Filters, and is the basis of all commands. For example, ./program | ./program2 puts all output of program 1 into program 2.

Note all redirectors can be chained, for example, ./program < lol.txt | ./program2 | ./program3 > out.txt. A chain of pipes is known as a pipeline.

Reading Files

cat

Takes unlimited arguments and outputs the contents of the files to standard output. This way, the files are concatenated together! For example, cat a.txt b.txt prints a.txt, then b.txt to standard output. This command can be useful for viewing a short file on screen temporarily.

If you do not give it an argument, it'll wait for standard input from the user, followed by the end of file character (CTRL+D) and prints that out.

more and less

More and less are both basic text viewers for terminal. Less is better than more, but both are functional. You can read and navigate a file by entering the command less a.txt. Less paginates output (displays it one page at a time), which is a very welcome feature when dealing with long files - you definitely do not what to simply cat a long file.

Text Editors

There are many available on UNIX (some of which you have to install), including gedit, emacs and vim. What you use today is up to you!
