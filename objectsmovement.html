Objects, Rooms and Movement

Objects

Sprites on their own are just images. If we want to use them in the game, we need to convert them into objects. Your game will consist of many objects such as your player, platforms and obstacles. Letâ€™s make the player object.

To create an object:

  1. Right click on the Objects folder and click Create Object (or click the sphere icon)
  2. Name your sprite obj_player
    Note: Again, we will use the naming convention obj_ at the start of the name so that we can easily tell that this is an object.
  3. Click on the drop down menu icon and select spr_player
  4. Check Visible and Solid

Try making a tile object!

Rooms

With a player and some tiles, we can now make a simple platform game! However, first we need a room to place them in.

To create an room:

  1. Right click on the Rooms folder and click Create Room (or click the rectangle icon)
  2. Click on the Objects tab in the Room Properties and select obj_tile in the drop down menu
  3. Left click on the grid to place the tiles down to make your floor. (Control + right click to delete)
  4. After you make your floor, change the Object to obj_player and place your player down once

Press F5 to test out your room. Does it feel like something is missing?

Horizontal Movement

Let's add player movement!

To implement movement to the left for your player:

  1. Right click on obj_player and click on properties (or double click obj_player)
  2. Click on the Add Event → Keyboard → <Left>
  3. Click on the Control tab and select the Check Empty icon (1st column, 1st row) and drag and drop it to the Actions window.
  4. Let x = -5, y = 0, objects = Only solid and check Relative
  5. Press OK
  6. Click on the Control tab and add a Start of a block (1st column, 4th row)
  7. Click on the Move tab and add a Jump to position action (1st column, 4th row)
    Let x = -5, y = 0 and check Relative
  8. Control tab → End of a block (1st column, 5th row)

    9. Your actions for the <Left> keyboard event should look like this so far.


    We're not done yet. So far the event will check if there's space left, and move left if that's the case. If there isn't space, we want to move left as much as possible. Or in other words, we want to move left until we are in contact with the tile.


  9. Click on the Control tab and add an Else (2nd column, 4th row)
  10. Add a Start of block
  11. Click on the Move tab and add a Move to contact action (1st column, 6th row)
    Let direction = 180, maximum = 5
    Note: direction is measured in degrees. A direction of 0 points strictly rightwards/eastwards and increasingly values move counter-clockwise. When direction = 180 the direction is strictly leftwards/westwards. Can you figure out which direction is indicated by direction = 270?
  12. Add an End of a block

Try creating a <Right> keyboard event!

Note: if you prefer, you can use WASD for movement instead of the arrow keys

Vertical Movement

We need to implement three things: gravity, landing and jumping.

To create gravity:

  1. Go to obj_player's properties
  2. Add Event → Step → Step
  3. Control tab → Check Empty icon (1st column, 1st row)
    Let y = 1 and check Relative
  4. Control tab → Start of a block icon (1st column, 4th row)
  5. Move tab → Set gravity icon (3rd column, 2nd row)
    Let direction = 270, gravity = 1
  6. Control tab → End of a block icon (1st column, 5th row)
  7. Control tab → Else icon (2nd column, 4th row)
  8. Control tab → Start of a block icon (1st column, 4th row)
  9. Move tab → Set gravity icon (3rd column, 2nd row)
    Let direction = 270, gravity = 0
  10. Control tab → End of a block icon (1st column, 5th row)

You should have something that looks like this.

Landing:

There may be issues such as the player falling and landing halfway through a tile. To fix this, we need to adjust the player's position so it's on top of the tile instead of inside it.

  1. Go to obj_tile's properties
  2. Add Event → Collision → obj_player
  3. Move tab → Move to contact icon (1st column, 6th row)
    Let direction = direction, maximum = 12, against = solid objects, applies to other
    Note: The applies to other radio button is extremely important. In this case, since we are under the Collision event "other" refers to the other object in this collision check (obj_player in this case) that isn't this object (which is obj_tile in this case).
  4. Move tab → Speed Vertical icon (2nd column, 2nd row) Let vert. speed = 0, applies to other

You should have something that looks like this.

Jumping:

We can add jumping by setting the player's vertical speed to -12. However we need to check if the player is standing on a tile before letting them jump.

  1. Go to obj_player's properties
  2. Add Event → Keyboard → <Up>
  3. Control tab → Check Empty icon
    Let x = 0, y = 1, object = Only solid, check Relative, check NOT// TODO
    Note: The NOT checkbox means that the command does the opposite of what it usually does. In this case we are saying "if the area 1 unit directly below the player is NOT empty of solid objects"
  4. Control tab → Start of a block
  5. Move tab → Speed Vertical
    Let vert. speed = -12
  6. Control tab → End of a block
